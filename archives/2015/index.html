:before和:after伪元素

:before :after作用：It creates a phoney element and inserts it before or after the content of the element that you’ve targeted.

1 #example:before {
2 content: “#”;
3 }
4
5 #example:after {
6 content: “.”;
7 }

  1. we’re targeting the same element using #example:before and #example:after. Strictly speaking, they’re the pseudo-elements in the code.

  2. Without the content property, pseudo-elements are useless. So while the pseudo-element selector itself is needed to target the element, you won’t be able to insert anything without adding the content property.

JavaScript高级程序设计读书笔记

原型对象：
无论什么时候，只要创建了一个新函数，就会根据一组特定的规则为该函数创建一个prototype属性，这个属性指向函数的原型对象。
在默认情况下，所有原型对象都会自动获得一个constructor（构造函数）属性，这个属性包含一个指向prototype属性所在函数的指针。
创建了自定义的构造函数之后，其原型对象默认只会取得constructor属性，至于其他方法，则都是从Object继承而来。
当调用构造函数创建一个新实例后，该实例的内部将包含一个指针（内部属性），指向构造函数的原型对象。ECMA-262第5版中管这个指针叫[[Prototype]]。这个连接存在于实例与构造函数的原型对象之间，而不是存在于实例与构造函数之间

CSS属性

margin: 10px 5px 15px; 表示：上外边距：10px，左右外边距：5px，下外边距：15px

margin: 10px 5px; 表示：上 下外边距：10px，左右外边距：5px

clear: none; 默认值，允许两边都可以有浮动对象
left; 不允许左边有浮动对象
right; 不允许右边有浮动对象
both; 两测不允许有浮动对象
